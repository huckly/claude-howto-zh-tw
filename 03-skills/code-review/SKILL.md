# 程式碼審查 Skill

此 skill 提供全面的程式碼審查能力，專注於：

1. **安全性分析**
   - 認證/授權問題
   - 資料暴露風險
   - 注入漏洞
   - 密碼學弱點
   - 敏感資料記錄

2. **效能審查**
   - 演算法效率（Big O 分析）
   - 記憶體最佳化
   - 資料庫查詢最佳化
   - 快取機會
   - 並行問題

3. **程式碼品質**
   - SOLID 原則
   - 設計模式
   - 命名慣例
   - 文件
   - 測試覆蓋率

4. **可維護性**
   - 程式碼可讀性
   - 函式大小（應 < 50 行）
   - 循環複雜度
   - 相依性管理
   - 型別安全

## 審查模板

對於每段被審查的程式碼，提供：

### 摘要
- 整體品質評估（1-5）
- 關鍵發現數量
- 建議的優先區域

### 嚴重問題（如有）
- **問題**：清楚的描述
- **位置**：檔案和行號
- **影響**：為什麼這很重要
- **嚴重度**：Critical/High/Medium
- **修復**：程式碼範例

### 依類別的發現

#### 安全性（如發現問題）
列出安全漏洞及範例

#### 效能（如發現問題）
列出效能問題及複雜度分析

#### 品質（如發現問題）
列出程式碼品質問題及重構建議

#### 可維護性（如發現問題）
列出可維護性問題及改善方案
