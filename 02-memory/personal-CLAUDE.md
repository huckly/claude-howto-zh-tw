# 我的開發偏好

## 關於我
- **經驗等級**：8 年全端開發
- **偏好語言**：TypeScript、Python
- **溝通風格**：直接，附帶範例
- **學習風格**：附帶程式碼的視覺圖表

## 程式碼偏好

### 錯誤處理
我偏好使用 try-catch 區塊和有意義的錯誤訊息進行明確的錯誤處理。
避免通用錯誤。始終記錄錯誤以便除錯。

### 註解
註解用於說明「為什麼」，而非「什麼」。程式碼應具有自我說明性。
註解應解釋業務邏輯或非顯而易見的決策。

### 測試
我偏好 TDD（測試驅動開發）。
先寫測試，再寫實作。
關注行為，而非實作細節。

### 架構
我偏好模組化、低耦合的設計。
使用依賴注入以提高可測試性。
關注點分離（Controllers、Services、Repositories）。

## 除錯偏好
- 使用帶有前綴的 console.log：`[DEBUG]`
- 包含上下文：函式名稱、相關變數
- 可用時使用堆疊追蹤
- 記錄中始終包含時間戳記

## 溝通
- 使用圖表解釋複雜概念
- 在解釋理論之前先展示具體範例
- 包含前後對照的程式碼片段
- 在結尾總結重點

## 專案組織
我以如下方式組織專案：
```
project/
  ├── src/
  │   ├── api/
  │   ├── services/
  │   ├── models/
  │   └── utils/
  ├── tests/
  ├── docs/
  └── docker/
```
