# 擴展單元測試

擴展現有的單元測試，使其符合專案的測試框架：

1. **分析涵蓋率**: 執行涵蓋率報告，以識別未測試的分支、邊緣案例和涵蓋率較低的區域
2. **識別缺口**: 審查程式碼，尋找邏輯分支、錯誤路徑、邊界條件、空值/空輸入
3. **撰寫測試**，使用專案的框架：
   - Jest/Vitest/Mocha (JavaScript/TypeScript)
   - pytest/unittest (Python)
   - Go testing/testify (Go)
   - Rust test framework (Rust)
4. **針對特定情境**:
   - 錯誤處理和例外狀況
   - 邊界值（最小值/最大值、空值、空）
   - 邊緣案例和角落案例
   - 狀態轉換和副作用
5. **驗證改善**: 再次執行涵蓋率，確認可衡量的提升

僅呈現新的測試程式碼區塊。遵循現有的測試模式和命名慣例。
